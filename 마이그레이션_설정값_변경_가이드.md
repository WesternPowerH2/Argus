# 마이그레이션 설정값 변경 가이드
## Google Apps Script & PowerShell 템플릿 활용법

**대상**: 새로운 Claude 계정/PC에서 마이그레이션을 진행할 담당자  
**목적**: 설정 템플릿 파일에서 정확히 어느 부분을 뭘로 변경해야 하는지 명확히 설명

## 🎯 마이그레이션 설정값 변경 방식

### 기존 방식 vs 새 방식

```
❌ 기존 (비효율적):
   현재 PC의 파일 복사 → 모든 설정값 재검색 → 코드 수정
   → 오류 가능성 높음

✅ 새 방식 (효율적):
   설정 템플릿 파일 사용 → 4개 항목만 변경 → 그대로 사용
   → 코드는 검증된 것 재사용, 설정값만 변경
```

### 2. PowerShell 스크립트 설정 (깃헙_자동업로드_설정_템플릿.ps1)

#### 파일 위치
```
새 PC의 경로: C:\Argus\깃헙_자동업로드.ps1
또는 다운로드: D:\OneDrive\문서\claude\Argus\Argus 수소정보지 정리\
             깃헙_자동업로드_설정_템플릿.ps1
```

#### 준비물
```
✓ GitHub Personal Access Token (CLAUDE.md Phase 1.2에서 생성)
✓ GitHub 계정명 (자동 인식되므로 일반적으로 변경 불필요)
✓ 저장소 이름 (기본값: argus)
```

#### 변경 과정

**Step 1: 파일 열기**
```
깃헙_자동업로드_설정_템플릿.ps1을 텍스트 에디터로 열기
(메모장, VS Code, PowerShell ISE 등)
```

**Step 2: 설정값 위치 확인**
```
파일의 처음 부분에 이 부분이 있습니다:

# ═══════════════════════════════════════════════════════════════════════════════
# 🔧 [설정값] — 이 3개 항목만 새 계정의 정보로 변경하면 됩니다
# ═══════════════════════════════════════════════════════════════════════════════

# 1️⃣ GitHub Personal Access Token (PAT)
$TOKEN  = "YOUR_GITHUB_PAT_HERE"

# 2️⃣ GitHub 계정 이름
$GITHUB_ACCOUNT = $null  # 자동 인식 (변경 금지)

# 3️⃣ GitHub 저장소 이름
$REPO   = "argus"
```

**Step 3: 각 설정값 변경**

##### 1️⃣ TOKEN 변경 (가장 중요)

```
변경 전:
$TOKEN  = "YOUR_GITHUB_PAT_HERE"

변경 후:
$TOKEN  = "ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"
           ↑ CLAUDE.md Phase 1.2에서 생성한 실제 GitHub PAT

확인:
  ✓ "ghp_" 로 시작
  ✓ 큰따옴표 유지 ("...")
  ✓ 공백 없음
  ✓ 전체 토큰 복사 (중간에 잘리면 안됨)

⚠️ 보안 주의:
  이 파일의 TOKEN은 노출되면 위험합니다.
  .gitignore에 자동 포함되므로 GitHub에 업로드되지 않습니다.
  하지만 로컬 PC에서 누군가 접근하면 위험하므로 주의하세요.
```

##### 2️⃣ GITHUB_ACCOUNT 변경 (변경 금지)

```
변경 전:
$GITHUB_ACCOUNT = $null  # 자동 인식 (변경 금지)

변경 후:
$GITHUB_ACCOUNT = $null  # ← 그대로 두세요!

이유:
  스크립트가 실행 중 GitHub API를 통해 자동으로 인식합니다.
  수동으로 입력하면 오히려 오류 발생 가능
```

##### 3️⃣ REPO 변경 (선택사항)

```
변경 전:
$REPO   = "argus"

변경 후 (저장소명 변경하고 싶으면):
$REPO   = "argus-h2-briefing"
또는
$REPO   = "hydrogen-briefing"

기본값 유지 권장:
$REPO   = "argus"

확인:
  ✓ 큰따옴표 유지 ("...")
  ✓ 영문자, 숫자, 하이픈(-) 만 사용
  ✓ 공백 없음
  ✓ 시작/끝에 하이픈 없음 (-argus 또는 argus- 불가)
```

**Step 4: 파일 저장**
```
Ctrl+S로 저장
인코딩: UTF-8 (또는 기본값)
```

**Step 5: 파일 배치**

```
저장한 파일을 다음 위치로 이동:

파일명 확인:
  ✓ 정확한 파일명: 깃헙_자동업로드.ps1
  ✗ 잘못된 예: 깃헙_자동업로드_설정_템플릿.ps1
```

## ⚠️ 자주 하는 실수

### Google Apps Script

| 실수 | 결과 | 해결 |
|------|------|------|
| 따옴표 빠뜨림: `const PDF_FOLDER_ID = 1oMyl4hT...` | 문법 오류 | 따옴표 추가: `'1oMyl4hT...'` |
| 세미콜론 제거: `const PDF_FOLDER_ID = '1oMyl...` | 문법 오류 | 세미콜론 추가: `'1oMyl...';` |
| 공백 추가: `const PDF_FOLDER_ID = ' 1oMyl...'` | 폴더 찾을 수 없음 | 공백 제거 |
| 이메일 틀림: `from:kim.sungho@naver.com` | 메일 감지 안됨 | 정확한 이메일 입력 |

### PowerShell 스크립트

| 실수 | 결과 | 해결 |
|------|------|------|
| 토큰 부분만 복사: `$TOKEN = "XXXXX"` (앞의 ghp_ 빠짐) | 인증 실패 | 전체 토큰 복사 |
| 큰따옴표 작은따옴표로 변경: `$TOKEN = 'ghp_...'` | 일반적으로 OK (PowerShell은 둘 다 허용) | 큰따옴표 권장 |
| GITHUB_ACCOUNT 수정: `$GITHUB_ACCOUNT = "MyName"` | 오류 발생 가능 | `$null` 유지 |
| 저장소명에 공백: `$REPO = "argus briefing"` | 저장소 생성 실패 | 공백 제거 또는 하이픈 사용 |

## 📝 변경 후 검증 방법

### Google Apps Script 검증

```
1. Google Apps Script 에디터 열기
2. 상단의 "Run" 버튼 클릭
3. 실행 로그 확인:
   
   ✅ 성공:
      "🔍 검색: from:new.person@example.com subject:Argus"
      "발견된 이메일: 1개" (또는 0개, 최근 메일이 없으면)
   
   ❌ 실패:
      "❌ 오류: 설정값이 완성되지 않았습니다."
      → 설정값을 다시 확인
      
      "❌ 오류: 폴더를 찾을 수 없습니다"
      → 폴더 ID 확인 (공백, 오타 등)
```

### PowerShell 스크립트 검증

```
1. PowerShell (관리자) 열기
2. 다음 명령 실행:
   powershell -ExecutionPolicy Bypass -File "C:\Argus\깃헙_자동업로드.ps1"

3. 출력 메시지 확인:
   
   ✅ 성공:
      "✅ 인증 완료: @[GitHub계정명]"
      "✅ N / N 개 파일 업로드 완료"
      "🌐 포털 주소: https://[계정].github.io/argus/"
   
   ❌ 실패:
      "❌ Authentication failed"
      → TOKEN 확인 (공백, 오타 등)
      
      "❌ 저장소를 찾을 수 없습니다"
      → REPO 이름 확인
```
